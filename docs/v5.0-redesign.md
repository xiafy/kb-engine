## 一、问题陈述

### 问题 1（v4.1 → v5.0 初版已解决）

v4.1 将 CRO 组织结构硬编码在训练引擎中（Clinical×3 + Biostat×1）。
**解决**：Owner Agent 架构——做题 Agent 读 SOP 自行决定编排方式。

### 问题 2（v5.0 初版 → v5.0 最终版）

v5.0 初版试图通过一套 program.md + config.yaml override 适配所有训练场景。
但 Protocol Design 和 SAP Design 的差异不是"配置层面"的，而是"逻辑层面"的：

| 维度 | Protocol Design | SAP Design |
|------|----------------|------------|
| 评分维度 | 10 项（终点、对照、入排…） | 完全不同（多重性、缺失数据、敏感性…） |
| 分析策略 | 临床决策差距（KNOW/REG） | 统计方法论差距（STAT/CALC） |
| 优化策略 | SOP 加临床规则 + 适应症知识 | SOP 加公式/方法论 + 工具使用 |
| 做题方式 | 临床设计为主 | 统计设计为主 |
| 题目提取 | Clinical Review | Statistical Review |

**解决**：拆分为 framework.md（不变骨架） + 每分支独立 program.md（可变细节）。

## 二、最终架构

### 目录结构

```
training/
├── _shared/
│   ├── framework.md          # 不变的流程骨架（Step 名 + 输入/输出契约）
│   ├── LANGUAGE.md            # 语言规范
│   ├── QUESTION-DESIGN.md     # 选题规范
│   ├── ROLES.md               # 角色规范
│   ├── SUPERSEDED.md          # 标记旧 program.md/evaluate.md 已弃用
│   ├── program.md             # [SUPERSEDED] 保留供参考
│   └── evaluate.md            # [SUPERSEDED] 保留供参考
│
├── protocol-design/
│   ├── program.md             # ✅ Protocol Design 完整训练程序 v5.0
│   ├── evaluate.md            # ✅ Protocol Design 评分标准（10 维度）
│   ├── case-config.json       # ✅ Protocol Design 案例配置
│   ├── config.yaml            # ✅ 分支配置 v2.0
│   ├── questions/             # 21 套题目
│   ├── results.tsv            # 训练记录
│   └── rounds/                # 每轮产出
│
└── sap-design/
    ├── program.md             # ⬜ 待创建（基于 framework.md 重写）
    ├── evaluate.md            # ⬜ 待创建（完全不同的评分维度）
    ├── case-config.json       # ⬜ 待创建
    ├── config.yaml
    └── ...
```

### framework.md 定义什么（不变）

只到 "Step 名称 + 输入/输出契约 + 全局不变量"：
- Step 0→8 的流程骨架
- 每步的输入/输出数据类型
- 6 条全局不变量（答案隔离、裁判隔离、Section-Append、偏差记录、Git PR、审核门控）

### 分支 program.md 定义什么（可变）

- 系统角色 prompt
- 评分维度和评分逻辑
- 做题 Agent 配置（数量、timeout、输入内容）
- 差异分析归类代码
- 规则溯源搜索范围
- 优化策略（怎么从差距提取规则）
- 分批策略和执行顺序
- 收敛判据参数
- 错误处理细节

### Owner Agent 架构（保留）

- 做题 Agent 读 SOP 自行决定是否 spawn sub-agent
- program.md 不规定 Owner Agent 内部编排
- SOP 中可包含编排指引（何时 spawn、给什么输入）
- 行为变异分析捕捉两个 Owner 的编排差异

## 三、v4.1 → v5.0 变更总结

| 方面 | v4.1 | v5.0 |
|------|------|------|
| 程序文件 | 1 个 program.md 管所有 | framework.md + 每分支独立 program.md |
| 评分标准 | 1 个 evaluate.md + branch override | 每分支独立 evaluate.md |
| 案例配置 | 1 个 _shared/case-config.json | 每分支独立 case-config.json |
| 做题 Agent | 硬编码 Clinical×3 + Biostat×1 | 2 Owner Agent，读 SOP 自行编排 |
| 角色定义 | program.md 中 | SOP 中 |
| 适用范围 | 试图通用但实际不通用 | 骨架通用，细节各分支自主 |
